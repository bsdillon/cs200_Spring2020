Assignment #4 - Tic-Tac-Toe

Purpose
Use GUI elements and background logic to determine a complex "win" condition.
Instructions
  1. Use a table to create a tic-tac-toe board. Put buttons in each <td> tag.
  2. When a button is pushed, call a function to remove the button from the tag and put an X.
  3. After placing an X, use whatever algorithm you want to place the O.
  4. After each X or O is placed you should check for a win. Remember there are sixteen different win conditions.
  5. Announce the winner if any exists.
  6. Add a reset button to restore the board to the starting position.
  7. Put all these elements in a page. Test to make sure the buttons work as expected and resolve any bugs. Consider the following:
    • Can the user click a button after the game ends? How can you prevent this from happening?
    • Can a button be pushed twice quickly? How can you prevent it?
    • What happens if no one wins?
    • Does the reset work before a game ends?
  8. Submit the link to the page in Canvas.
