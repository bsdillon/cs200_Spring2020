Homework 2 - Developer tools and code walkthrough

This homework will work with an auxiliary file that you will learn from. Follow the link and save that text as an HTML file. Run it in a browser and you should see a long table of values.

  1. Open up the developer tools for your browser. For Chrome it is CTRL+Shift+I. For other browsers it should be easy enough to look it up online. This is part of what we refer to as the developer environment. In most programming languages today you can get one tool that does all these things AND allows you to edit code. This is known as an integrated development environment or IDE.
  2. In this image you can see all the parts of the developer tools as they appear in Chrome. Even if your browser is different the same parts should exist. Find them all.

    1. The file panel. Some pages refer to multiple files and you can select whichever one you want to look at in the tools.
    2. Sources. Most often you will want to look at the source code associated with a file. There are other tabs, but this is the most important.
    3. Console. For Javascript you can write some things to the console and that is one way of debugging your program. I prefer to write to the screen, but for some projects there is no screeen. The console always exists.
    4. Watch. These are variables you want to see. Hit the + on the right. Add variables x, y, z, and i to your watch list just like it shows here.
    5. Call stack. When your code has multiple function calls it helps to see where you are in that long list of function calls. The call stack shows that. Ever heard of a "stack overflow"? This is where that happesn.
    6. This is a breakpoint. You can add one by clicking on the number of that line or click on it again to remove it. This is where the program will stop and let you look around.
    7. Breakpoints. A good IDE gives you a list of all breakpoints, just in case you get lost.
  3. Add a breakpoint on the lines where it says AddRow. Refresh the page and the program will stop at the first breakpoint on line 14. You can walk through the program using these buttons.

    1. Next BreakPoint. This will start the code running again until it reaches the next break point (or to the end if there are no other break points). So if you are on line a it will stop again at line c.
    2. Step over. The next line might lead to something you don't want to follow, so we can jump it. Again, if you are on line b it will skip the doThis function and go to line c.
    3. Step into. The next line may be a function that we want to follow. If you are on line b this will take you to line d.
    4. Step forward. This will go to the next line following whatever function is called or not. Just pushing this button from line a will take to to b, d, e, and then c in that order.
    5. Step out. Maybe you are in a function and want to get out without going trough every line. This goes right out of the function. If you are on line d it will skip back to c.
  4. Use Step forward to walk through the whole program one time. At each break point, read the watch variables and predict what text will be added to the page.
  5. Examine the if-else statement on line 21. You can read more about them at w3schools. Run through the program again and predict whether label 3a or 3b will follow label 2.
  6. Examine the for loop on line 16. Again, you can read more about them at w3schools. Learn how the three statements work. Run through the program again and predict how the program will run each time it gets to line 16.
  7. Examine the while loop on line 34. Read about them at w3schools. Run the program through again and predict how the program will run each time it gets to line 34.
  8. Now that you are familiar with the use of developer tools, you can use them in all your future projects. When you test your programs you can put a break point at important places and step through spaces that you want to check.
