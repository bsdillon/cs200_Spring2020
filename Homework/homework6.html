Homework 6 - GoldField: Looking at locations

This is the first homework that will use the goldfield API. If you have any questions, you can read more about the API at that link. It was created expressly for the purpose of teaching programming by simplifying certain actions and allowing a student to get more done while working only with the most basic parts of the code. In this homework assignment you will work with the LookAt function.

  1. Create a web page. Put on a p or div tag with an id. In the head add a link to the goldfield API. This link will add all the code from the goldfield API just as if you had written it into your webpage.
    <script src="https://bsdillon.github.io/cs200_Spring2020/GoldField/goldfield.js"></script>
  2. Test the goldfield API. In your page's script, create a new GoldField. Give it the tag you created in the last step as the location for the GoldField. You should see a little green field with nothing on it. Experiment with the size of the parameters for the field to create one that is very wide, but not very tall.
    var field = new GoldField(document.getElementById("YOUR TAG ID HERE"),BLOCK SIZE,BLOCKS WIDE,BLOCKS HIGH);
  3. Next we will experiment with adding Movers to the field. Add code to put on one of the built in Mover types. Each one has its own AddMOVER method. For instance, there is an AddTower() function that puts a tower in a random spot on the screen. Use the field variable to adding one each of these types to GoldField created in the previous step:
    • Hero
    • Tower
    • Monster
    • Gold
    • Rock
    • Water
    • Tree
    4. You can also add obstacles randomly to the scene. Add one obstacle randomly using AddObstacle().
    5. Although a Mover is put on the screen randomly. You can put it wherever you want. When you created the Hero, you probably didn't save the hero into a variable. Change that code to create a variable just like the one we did for field. Then use that variable to SetPosition. Put the hero on the screen first and put him at (0,0).
  5. Finally you can add a large group of obstacles randomly to fill the field. Try RandomFill. Try filling to 26%. What happens? Pick a percent that makes the field look reasonably full to you.
